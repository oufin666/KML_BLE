# 按键模块移植说明

## 1. 功能概述

按键模块实现了对KEYB和KEYC两个按键的检测功能，主要包括：

- 按键初始化
- 按键状态读取（按下/释放）
- 按键抬起事件检测
- 支持轮询方式检测按键状态

## 2. 需要移植的文件

### 2.1 新增文件

| 文件路径 | 说明 |
|---------|------|
| `Drivers/BSP/inc/KEY.h` | 按键模块头文件 |
| `Drivers/BSP/src/KEY.c` | 按键模块实现文件 |

### 2.2 需要修改的文件

| 文件路径 | 说明 |
|---------|------|
| `Core/Src/freertos.c` | 在任务中添加按键检测代码 |

## 3. 移植步骤

### 3.1 复制文件

1. 将 `KEY.h` 复制到目标项目的 `Drivers/BSP/inc/` 目录
2. 将 `KEY.c` 复制到目标项目的 `Drivers/BSP/src/` 目录

### 3.2 配置GPIO引脚

在CubeMX中配置以下GPIO引脚：

| 引脚 | 模式 | 配置 | 说明 |
|------|------|------|------|
| PB14 | GPIO_Input | Pull-up | KEYB 按键B |
| PB13 | GPIO_Input | Pull-up | KEYC 按键C |

**配置步骤**：

1. **进入GPIO配置**：
   - 左侧菜单 → System Core → GPIO
   - 或直接在引脚视图中点击 PB13 和 PB14

2. **配置PB13（KEYC）**：
   - 点击 PB13 引脚
   - **GPIO mode**: GPIO_Input
   - **GPIO Pull-up/Pull-down**: Pull-up
   - **Maximum output speed**: Low
   - **User Label**: KEYC

3. **配置PB14（KEYB）**：
   - 点击 PB14 引脚
   - **GPIO mode**: GPIO_Input
   - **GPIO Pull-up/Pull-down**: Pull-up
   - **Maximum output speed**: Low
   - **User Label**: KEYB

### 3.3 修改 freertos.c

在任务文件中添加按键检测代码：

1. **添加头文件包含**：
   ```c
   #include "../../Drivers/BSP/inc/KEY.h"
   ```

2. **在任务初始化部分添加按键初始化**：
   ```c
   // 初始化按键
   KEY_Init();
   sprintf(tx_buffer, "[KEY] Initialization success!\r\n");
   HAL_UART_Transmit(BLE_UART_HANDLE, (uint8_t*)tx_buffer, strlen(tx_buffer), 100);
   ```

3. **在任务主循环中添加按键状态更新**：
   ```c
   // 每10毫秒更新一次按键状态
   KEY_Update();
   ```

4. **在数据发送部分添加按键状态检测**：
   ```c
   // 更新按键状态
   KEY_Update();
   
   // 检测按键抬起事件
   keyb_released = KEYB_HasReleased();
   keyc_released = KEYC_HasReleased();
   
   // 发送姿态角、翻腕标志位和按键状态
   sprintf(tx_buffer, "pitch:%6.2f  FLAG：%d  KEYB：%d  KEYC:%d\r\n", 
           pitch, wrist_raise_flag, keyb_released, keyc_released);
   HAL_UART_Transmit(BLE_UART_HANDLE, (uint8_t*)tx_buffer, strlen(tx_buffer), 100);
   ```

## 4. 函数说明

### 4.1 按键函数

| 函数名 | 说明 | 参数 | 返回值 |
|-------|------|------|--------|
| `KEY_Init` | 初始化按键 | 无 | HAL_StatusTypeDef 初始化状态 |
| `KEYB_Read` | 读取KEYB按键状态 | 无 | uint8_t 按键状态：KEY_PRESSED(0)或KEY_RELEASED(1) |
| `KEYC_Read` | 读取KEYC按键状态 | 无 | uint8_t 按键状态：KEY_PRESSED(0)或KEY_RELEASED(1) |
| `KEYB_IsPressed` | 检查KEYB是否被按下 | 无 | uint8_t 1: 按键被按下，0: 按键未被按下 |
| `KEYC_IsPressed` | 检查KEYC是否被按下 | 无 | uint8_t 1: 按键被按下，0: 按键未被按下 |
| `KEYB_IsReleased` | 检查KEYB是否处于释放状态 | 无 | uint8_t 1: 按键已释放，0: 按键未释放 |
| `KEYC_IsReleased` | 检查KEYC是否处于释放状态 | 无 | uint8_t 1: 按键已释放，0: 按键未释放 |
| `KEYB_HasReleased` | 检查KEYB是否刚刚抬起 | 无 | uint8_t 1: 按键刚刚抬起，0: 按键未抬起 |
| `KEYC_HasReleased` | 检查KEYC是否刚刚抬起 | 无 | uint8_t 1: 按键刚刚抬起，0: 按键未抬起 |
| `KEY_Update` | 更新按键状态，检测按键抬起事件 | 无 | 无 |

## 5. 使用方法

### 5.1 基本使用

```c
// 初始化按键
KEY_Init();

// 主循环中
while (1) {
    // 更新按键状态
    KEY_Update();
    
    // 检查按键抬起事件
    if (KEYB_HasReleased()) {
        // KEYB按键刚刚抬起
        // 处理按键抬起事件
        printf("KEYB released\r\n");
    }
    
    if (KEYC_HasReleased()) {
        // KEYC按键刚刚抬起
        // 处理按键抬起事件
        printf("KEYC released\r\n");
    }
    
    // 其他代码...
    osDelay(10);
}
```

### 5.2 按键状态检测

```c
// 初始化按键
KEY_Init();

// 主循环中
while (1) {
    // 更新按键状态
    KEY_Update();
    
    // 检查按键当前状态
    if (KEYB_IsPressed()) {
        // KEYB按键当前被按下
        printf("KEYB is pressed\r\n");
    }
    
    if (KEYC_IsReleased()) {
        // KEYC按键当前处于释放状态
        printf("KEYC is released\r\n");
    }
    
    // 其他代码...
    osDelay(50);
}
```

## 6. 注意事项

1. **GPIO配置**：
   - 确保PB13和PB14引脚已正确配置为GPIO输入模式
   - 必须启用内部上拉电阻（Pull-up）

2. **按键检测方式**：
   - 采用轮询方式检测按键状态，而非外部中断
   - 建议每10-20毫秒更新一次按键状态
   - 这种方式在低频率检测场景下比中断更节省资源

3. **按键状态**：
   - 按键按下时返回0（KEY_PRESSED）
   - 按键释放时返回1（KEY_RELEASED）
   - 这是因为按键通过上拉电阻连接，按下时接地

4. **抬起事件检测**：
   - `KEYB_HasReleased()` 和 `KEYC_HasReleased()` 函数会返回按键是否刚刚抬起
   - 这些函数会自动清除标志，因此每次调用只会返回一次抬起事件

## 7. 故障排除

- **按键无响应**：
  - 检查PB13和PB14引脚连接
  - 确认GPIO配置是否正确（Input模式，Pull-up）
  - 检查按键硬件连接是否正常

- **按键状态不稳定**：
  - 检查按键是否存在机械抖动
  - 可以在代码中添加软件去抖逻辑
  - 确保按键硬件连接稳定

- **抬起事件不触发**：
  - 确保 `KEY_Update()` 函数被定期调用
  - 检查按键按下和释放的完整过程是否被捕获
