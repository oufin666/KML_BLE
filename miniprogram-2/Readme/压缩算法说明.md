# KML差分压缩算法说明（硬件端解压指南）

## 目标

硬件端接收压缩数据后，解压得到**转换后的bin文件**（不是原始KML文件）。

bin文件格式与PC端解析器兼容，可直接用于导航和轨迹显示。

## 算法概述

采用**差分压缩算法**，利用相邻坐标点的相似性：
- 第一个点：完整存储（12字节：经度4字节 + 纬度4字节 + 海拔4字节）
- 后续点：只存储与前一个点的差值（约1-3字节/坐标）
- 压缩率：约75%（从12字节/点压缩到3字节/点）
- 坐标精度：保持6位小数（乘以10^6转为int32）
- 段点数：保持与原文件一致，不重新计算

## 压缩后的bin文件格式（硬件端需要解压出的目标格式）

```
[总段数 4字节] [段1点数 4字节] [段1点数据...] [段2点数 4字节] [段2点数据...] ...
```

### 详细说明

**文件头：**
- 总段数（4字节，uint32，小端序）

**每个段：**
- 段点数（4字节，uint32，小端序）
- 点数据（压缩格式）：
  - 第一个点（12字节）：
    - 经度int32（4字节，小端序）= 经度 × 1,000,000
    - 纬度int32（4字节，小端序）= 纬度 × 1,000,000  
    - 海拔int32（4字节，小端序）= 海拔 × 1,000,000
  - 后续点（可变长度，1-9字节/点）：
    - 经度差值（可变长度编码）
    - 纬度差值（可变长度编码）
    - 海拔差值（可变长度编码）

### 示例

假设有1个段，包含3个点：
```
01 00 00 00              // 总段数 = 1
03 00 00 00              // 段1点数 = 3
[第1个点 12字节]         // 完整坐标
[第2个点 3字节]          // 差值编码
[第3个点 3字节]          // 差值编码
```

## 差值编码格式（核心算法）

### 字节格式

**第一个字节：**
```
位7: 继续位 (1=有后续字节, 0=结束)
位6: 符号位 (1=负数, 0=正数)
位5-0: 数据位 (6位)
```

**后续字节：**
```
位7: 继续位 (1=有后续字节, 0=结束)
位6-0: 数据位 (7位)
```

### 编码示例

| 差值 | 编码结果 | 说明 |
|------|---------|------|
| 0 | `0x00` | 1字节 |
| 10 | `0x0A` | 1字节 |
| -10 | `0x4A` | 1字节（符号位=1） |
| 100 | `0x81 0x01` | 2字节（100 = 64×1 + 36） |
| -100 | `0xC1 0x01` | 2字节（符号位=1） |
| 1000 | `0x88 0x0F` | 2字节 |

### 编码算法（伪代码）

```c
// 编码差值
uint8_t* encode_delta(int32_t delta, int* out_len) {
    static uint8_t buffer[5]; // 最多5字节
    int len = 0;
    
    // 处理符号
    uint8_t sign = (delta < 0) ? 1 : 0;
    uint32_t abs_delta = (delta < 0) ? -delta : delta;
    
    // 第一个字节：6位数据
    uint8_t first_byte = abs_delta & 0x3F;
    abs_delta >>= 6;
    
    if (abs_delta > 0) first_byte |= 0x80;  // 继续位
    if (sign) first_byte |= 0x40;           // 符号位
    
    buffer[len++] = first_byte;
    
    // 后续字节：每字节7位数据
    while (abs_delta > 0) {
        uint8_t byte = abs_delta & 0x7F;
        abs_delta >>= 7;
        if (abs_delta > 0) byte |= 0x80;    // 继续位
        buffer[len++] = byte;
    }
    
    *out_len = len;
    return buffer;
}
```

### 解码算法（伪代码）

```c
// 解码差值
int32_t decode_delta(const uint8_t* bytes, int* bytes_consumed) {
    int32_t result = 0;
    uint8_t sign = 0;
    int shift = 0;
    int i = 0;
    
    while (1) {
        uint8_t byte = bytes[i];
        
        // 第一个字节：提取符号位
        if (i == 0) {
            sign = (byte & 0x40) >> 6;
        }
        
        // 提取数据位
        uint8_t data_bits = (i == 0) ? (byte & 0x3F) : (byte & 0x7F);
        
        // 累加数据（低位优先）
        result |= (data_bits << shift);
        shift += (i == 0) ? 6 : 7;
        
        i++;
        
        // 检查继续位
        if ((byte & 0x80) == 0) break;
    }
    
    *bytes_consumed = i;
    return sign ? -result : result;
}
```

## 硬件端解压算法（C语言实现）

### 完整解压函数

```c
#include <stdint.h>
#include <string.h>

/**
 * 解码差值
 * @param bytes 编码的字节数组
 * @param bytes_consumed 输出：消耗的字节数
 * @return 解码后的差值
 */
int32_t decode_delta(const uint8_t* bytes, int* bytes_consumed) {
    int32_t result = 0;
    uint8_t sign = 0;
    int shift = 0;
    int i = 0;
    
    while (1) {
        uint8_t byte = bytes[i];
        
        // 第一个字节：提取符号位
        if (i == 0) {
            sign = (byte & 0x40) >> 6;
        }
        
        // 提取数据位
        uint8_t data_bits = (i == 0) ? (byte & 0x3F) : (byte & 0x7F);
        
        // 累加数据（低位优先）
        result |= (data_bits << shift);
        shift += (i == 0) ? 6 : 7;
        
        i++;
        
        // 检查继续位
        if ((byte & 0x80) == 0) break;
    }
    
    *bytes_consumed = i;
    return sign ? -result : result;
}

/**
 * 读取int32（小端序）
 */
int32_t read_int32_le(const uint8_t* data) {
    return (int32_t)(data[0] | (data[1] << 8) | (data[2] << 16) | (data[3] << 24));
}

/**
 * 写入int32（小端序）
 */
void write_int32_le(uint8_t* data, int32_t value) {
    data[0] = value & 0xFF;
    data[1] = (value >> 8) & 0xFF;
    data[2] = (value >> 16) & 0xFF;
    data[3] = (value >> 24) & 0xFF;
}

/**
 * 读取uint32（小端序）
 */
uint32_t read_uint32_le(const uint8_t* data) {
    return (uint32_t)(data[0] | (data[1] << 8) | (data[2] << 16) | (data[3] << 24));
}

/**
 * 写入uint32（小端序）
 */
void write_uint32_le(uint8_t* data, uint32_t value) {
    data[0] = value & 0xFF;
    data[1] = (value >> 8) & 0xFF;
    data[2] = (value >> 16) & 0xFF;
    data[3] = (value >> 24) & 0xFF;
}

/**
 * 解压bin文件
 * @param compressed 压缩数据
 * @param compressed_len 压缩数据长度
 * @param decompressed 解压缓冲区
 * @param decompressed_max_len 解压缓冲区最大长度
 * @return 解压后的数据长度，失败返回-1
 */
int decompress_bin(const uint8_t* compressed, int compressed_len, 
                   uint8_t* decompressed, int decompressed_max_len) {
    int src_offset = 0;
    int dst_offset = 0;
    
    // 读取总段数
    if (compressed_len < 4) return -1;
    uint32_t total_segments = read_uint32_le(&compressed[src_offset]);
    src_offset += 4;
    
    // 写入总段数
    if (dst_offset + 4 > decompressed_max_len) return -1;
    write_uint32_le(&decompressed[dst_offset], total_segments);
    dst_offset += 4;
    
    // 处理每个段
    for (uint32_t seg = 0; seg < total_segments; seg++) {
        // 读取段点数
        if (src_offset + 4 > compressed_len) return -1;
        uint32_t point_count = read_uint32_le(&compressed[src_offset]);
        src_offset += 4;
        
        // 写入段点数
        if (dst_offset + 4 > decompressed_max_len) return -1;
        write_uint32_le(&decompressed[dst_offset], point_count);
        dst_offset += 4;
        
        if (point_count == 0) continue;
        
        // 读取第一个点（完整坐标）
        if (src_offset + 12 > compressed_len) return -1;
        int32_t prev_lon = read_int32_le(&compressed[src_offset]);
        int32_t prev_lat = read_int32_le(&compressed[src_offset + 4]);
        int32_t prev_alt = read_int32_le(&compressed[src_offset + 8]);
        src_offset += 12;
        
        // 写入第一个点
        if (dst_offset + 12 > decompressed_max_len) return -1;
        write_int32_le(&decompressed[dst_offset], prev_lon);
        write_int32_le(&decompressed[dst_offset + 4], prev_lat);
        write_int32_le(&decompressed[dst_offset + 8], prev_alt);
        dst_offset += 12;
        
        // 解压后续点
        for (uint32_t i = 1; i < point_count; i++) {
            // 解码经度差值
            int bytes_consumed;
            int32_t delta_lon = decode_delta(&compressed[src_offset], &bytes_consumed);
            src_offset += bytes_consumed;
            
            // 解码纬度差值
            int32_t delta_lat = decode_delta(&compressed[src_offset], &bytes_consumed);
            src_offset += bytes_consumed;
            
            // 解码海拔差值
            int32_t delta_alt = decode_delta(&compressed[src_offset], &bytes_consumed);
            src_offset += bytes_consumed;
            
            // 计算当前点坐标
            int32_t curr_lon = prev_lon + delta_lon;
            int32_t curr_lat = prev_lat + delta_lat;
            int32_t curr_alt = prev_alt + delta_alt;
            
            // 写入当前点
            if (dst_offset + 12 > decompressed_max_len) return -1;
            write_int32_le(&decompressed[dst_offset], curr_lon);
            write_int32_le(&decompressed[dst_offset + 4], curr_lat);
            write_int32_le(&decompressed[dst_offset + 8], curr_alt);
            dst_offset += 12;
            
            // 更新前一个点
            prev_lon = curr_lon;
            prev_lat = curr_lat;
            prev_alt = curr_alt;
        }
    }
    
    return dst_offset;
}
```

### 蓝牙数据接收处理

```c
#define COMPRESSED_BUFFER_SIZE 8192
#define DECOMPRESSED_BUFFER_SIZE 16384

static uint8_t compressed_buffer[COMPRESSED_BUFFER_SIZE];
static int compressed_offset = 0;
static uint8_t decompressed_buffer[DECOMPRESSED_BUFFER_SIZE];

/**
 * 处理接收到的蓝牙数据包
 * @param data 数据包
 * @param len 数据包长度
 */
void handle_ble_packet(const uint8_t* data, int len) {
    // 检查缓冲区空间
    if (compressed_offset + len > COMPRESSED_BUFFER_SIZE) {
        printf("Error: Buffer overflow\n");
        compressed_offset = 0;
        return;
    }
    
    // 追加数据到缓冲区
    memcpy(&compressed_buffer[compressed_offset], data, len);
    compressed_offset += len;
    
    // 检查是否收到结束符序列 0xFF 0xFF 0xFF
    if (len >= 3 && 
        data[len - 1] == 0xFF && 
        data[len - 2] == 0xFF && 
        data[len - 3] == 0xFF) {
        
        printf("Received end marker, decompressing...\n");
        
        // 移除结束符序列
        int compressed_data_len = compressed_offset - 3;
        
        // 解压数据
        int decompressed_len = decompress_bin(
            compressed_buffer, 
            compressed_data_len,
            decompressed_buffer, 
            DECOMPRESSED_BUFFER_SIZE
        );
        
        if (decompressed_len > 0) {
            printf("Decompression successful: %d bytes\n", decompressed_len);
            
            // 保存到文件或进一步处理
            save_bin_file(decompressed_buffer, decompressed_len);
        } else {
            printf("Decompression failed\n");
        }
        
        // 重置缓冲区
        compressed_offset = 0;
    }
}

/**
 * 保存bin文件到存储器
 */
void save_bin_file(const uint8_t* data, int len) {
    // 这里实现文件保存逻辑
    // 例如：保存到SD卡、Flash等
    printf("Saving bin file: %d bytes\n", len);
    
    // 示例：打印文件头信息
    if (len >= 4) {
        uint32_t total_segments = read_uint32_le(data);
        printf("Total segments: %u\n", total_segments);
    }
}
```

## 使用解压后的bin文件

解压后的bin文件可以直接用于导航和轨迹显示，格式与PC端解析器完全兼容。

### 读取bin文件示例

```c
/**
 * 读取并解析bin文件
 */
void parse_bin_file(const uint8_t* bin_data, int bin_len) {
    int offset = 0;
    
    // 读取总段数
    if (bin_len < 4) return;
    uint32_t total_segments = read_uint32_le(&bin_data[offset]);
    offset += 4;
    
    printf("Total segments: %u\n", total_segments);
    
    // 遍历每个段
    for (uint32_t seg = 0; seg < total_segments; seg++) {
        // 读取段点数
        if (offset + 4 > bin_len) return;
        uint32_t point_count = read_uint32_le(&bin_data[offset]);
        offset += 4;
        
        printf("Segment %u: %u points\n", seg + 1, point_count);
        
        // 读取每个点
        for (uint32_t i = 0; i < point_count; i++) {
            if (offset + 12 > bin_len) return;
            
            int32_t lon_int = read_int32_le(&bin_data[offset]);
            int32_t lat_int = read_int32_le(&bin_data[offset + 4]);
            int32_t alt_int = read_int32_le(&bin_data[offset + 8]);
            offset += 12;
            
            // 转换为浮点数
            float lon = lon_int / 1000000.0f;
            float lat = lat_int / 1000000.0f;
            float alt = alt_int / 1000000.0f;
            
            printf("  Point %u: lon=%.6f, lat=%.6f, alt=%.2f\n", 
                   i + 1, lon, lat, alt);
        }
    }
}
```

## 重要注意事项

### 1. 数据传输
- 蓝牙传输时，最后会收到结束符序列 `0xFF 0xFF 0xFF`（单独一包）
- 解压前必须移除这3个字节的结束符
- 分包大小：230字节/包，发送间隔：20ms

### 2. 字节序
- 所有多字节数据（uint32、int32）均使用**小端序**（Little-Endian）
- 示例：0x12345678 存储为 `78 56 34 12`

### 3. 内存需求
- 压缩数据缓冲区：建议8KB
- 解压数据缓冲区：建议16KB（约为压缩数据的2倍）
- 实际需求取决于KML文件大小

### 4. 坐标精度
- 坐标值乘以1,000,000后存储为int32
- 支持正负坐标（有符号整数）
- 精度：6位小数

### 5. 数据格式一致性
- 压缩前后的bin文件格式保持一致
- 段点数保持与原文件一致，不重新计算
- 坐标数据均使用int32格式存储
- 解压后的文件与原始bin文件完全相同

### 6. 错误处理
- 检查缓冲区溢出
- 验证数据长度
- 处理解压失败情况

5. **验证坐标**：检查坐标值是否在合理范围内

## 常见问题

### Q1: 解压后数据长度不对？
**A:** 检查是否正确移除了结束符序列（3字节）

### Q2: 坐标值异常？
**A:** 检查字节序是否正确（应使用小端序）

### Q3: 解压失败返回-1？
**A:** 检查压缩数据是否完整，缓冲区是否足够大

### Q4: 如何验证解压正确？
**A:** 对比解压后的bin文件大小，应该比压缩数据大约4倍
